Issue NVM Commands

Last updated: 2023/10/29

Namespace nvme::ioc

The functions which are devised to issue NVM commands are called with nvme::ioc namespace resolution.

IO Submission Queue Handle

To issue an NVM command to the NVMe SSD, the IO Submission Queue handle should be specified.

To issue an NVM command to the NVMe SSD through IO Submission Queue 1, the handle of the IO Submission Queue 1 should be specified to the function.

To issue an NVM command through Submission Queue 2, the handle of the IO Submission Queue 2 should be specified to the function.

Command Handle

After the function has issued an NVM command to the NVMe SSD, a command handle is returned by the function.

pspp()

The pspp() is used to poll the status of the command issued.

Example 1

This example shows how to issue a read command to the NVMe SSD. This example assumes that the namespace does not support metadata.

The 'struct funcInfo_t' and 'struct namespace_t' are structures implemented in script to ease retrieve controller (NVMe controller, or PCI function) information and information of NVMe namespaces. 

      
    struct funcInfo_t funcInfo;
    struct namespace_t ns;

    funcHandle = nvme::getFuncHandle(EC, 0);
    nvme::init(EC, funcHandle);

    funcInfo.Invalidate();
    funcInfo.RetrieveInfo(0, funcHandle);
    if (funcInfo.GetActiveNSIdNum() <= 0) {
        c::printf("No namespace is available!\n\n");
        goto TestEnd;
    }

    funcInfo.CopyTheNthNamespace(0, ns);

    sq1Handle = nvme::getSQHandle(EC, funcHandle, 1);

    nLBA = 0;
    nLen = 4;
    bufHandle = nvme::ddbAlloc(EC, funcHandle, ns.GetLBADataSize_byte() * nLen);

    do 
        cmdHandle = nvme::ioc::read(EC, sq1Handle, bufHandle, nLBA, nLen);
    while (cmdHandle == 0);

    while (1) {
        stsVal = nvme::pspp(cmdHandle);
        if (stsVal)
            break;
    }

    if (stsVal & nvme::cmdCplFlag) {
        nvme::ddbDump(EC, bufHandle, 0, 0);
    }
    else {
        ...
    }

Example 2

This example shows how to issue a write command to the NVMe SSD whose namespace supports metadata and is formatted to transmit metadata immediately after each LBA data.

The value 0xABCDABCD is filled into the first 4 bytes of the metadata of each LBA. 

      
    struct funcInfo_t funcInfo;
    struct namespace_t ns;

    funcHandle = nvme::getFuncHandle(EC, 0);
    nvme::init(EC, funcHandle);

    funcInfo.Invalidate();
    funcInfo.RetrieveInfo(0, funcHandle);
    if (funcInfo.GetActiveNSIdNum() <= 0) {
        c::printf("No namespace is available!\n\n");
        goto TestEnd;
    }

    funcInfo.CopyTheNthNamespace(0, ns);

    sq1Handle = nvme::getSQHandle(EC, funcHandle, 1);

    LBADataSize = ns.GetLBADataSize_byte();
    MetadataSize = ns.GetMetadataSize_byte();
    ExtLBADataSize = ns.GetExtLBADataSize_byte();

    nLBA = 0;
    nLen = 4;

    if (LBADataSize < ExtLBADataSize) {
        bufHandle = nvme::ddbAlloc(EC, funcHandle, ExtLBADataSize * nLen);
        for (i = 1; i <= nLen; ++i) {
            nvme::ddbWriteInt32(EC, bufHandle, i * ExtLBADataSize - MetadataSize, 1, 0xABCDABCD);
            ...
        }
    }
    else
        bufHandle = nvme::ddbAlloc(EC, funcHandle, LBADataSize * nLen);

    do 
        cmdHandle = nvme::ioc::write(EC, sq1Handle, bufHandle, nLBA, nLen);
    while (cmdHandle == 0);

    while (1) {
        stsVal = nvme::pspp(cmdHandle);
        if (stsVal)
            break;
    }
