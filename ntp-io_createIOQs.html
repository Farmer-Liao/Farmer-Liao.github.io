Create IO Submission/Completion Queues

Last updated: 2023/10/17

Use init()

IO submission queues and completion queues can be created by the init() function.

Example 1

This example script calls init() with only 2 arguments, which wants init() function create IO submission queues and completion queues by a default setting. Usually 4 IO submission queues and 1 IO completion queue are created. After IO queues are created, the number of IO submission queues can be known by getIOSQNum() function. Next, each queue's handle can be got by getSQHandle(). With the queue handle, the script can then submit NVM commands. 

      
    funcHandle = nvme::getFuncHandle(EC, 0);
    nvme::init(EC, funcHandle);
    iosqnum = nvme::getIOSQNum(EC, funcHandle);
    for (sqid = 1; sqid <= iosqnum; ++sqid) {
        sqhandle = nvme::getSQHandle(EC, funcHandle, sqid);
        ...
    }
Example 2

This example script calls init() with all 3 arguments, which wants init() function create IO submission queues and completion queues by the specified settings. In this example, it wants to create 32 IO Submission queues with size 128; also create 1 completion queue with size 512 for every 4 IO Submission queues. There are 7 parameters that can be specified in nvme::queueInitParams_t structure. The init() function will apply default settings for not-specified parameters.

Note that the real number of queues and real size of queues may be different from the specified, since the init() function would peek the SSD's capabilities and does adaptions. 

      
    struct nvme::queueInitParams_t qInitParams;

    funcHandle = nvme::getFuncHandle(EC, 0);
    qInitParams.setDef();
    qInitParams.nIOSQNum = 32;
    qInitParams.nIOSQSize = 128;
    qInitParams.nIOSQNumPerIOCQ = 4;
    qInitParams.nIOCQSize = 512;
    nvme::init(EC, funcHandle, qInitParams);
    iosqnum = nvme::getIOSQNum(EC, funcHandle);
    for (sqid = 1; sqid <= iosqnum; ++sqid) {
        sqhandle = nvme::getSQHandle(EC, funcHandle, sqid);
        ...
    }

Use createIOSQ() and createIOCQ()

Another way to create IO queues is to issue "Create I/O Submission Queue" and "Create I/O Completion Queue" commands. Before that, the Admin submission queue and completion queue should be created, and it was achieved by the initA() function. 

      
    funcHandle = nvme::getFuncHandle(EC, 0);
    nASQSize = nACQSize = 512;
    nvme::initA(EC, funcHandle, nASQSize, nACQSize);
    nMaxIOCQNum = nvme::getMaxIOCQNum(EC, funcHandle);
    nMaxIOSQNum = nvme::getMaxIOSQNum(EC, funcHandle);
    nMaxIOQSize = nvme::getMaxIOQSize(EC, funcHandle);
    cmdHandle = nvme::admc::createIOCQ(EC, funcHandle, cqid=1, 256);
    // poll command complete
    cmdHandle = nvme::admc::createIOSQ(EC, funcHandle, sqid=1, cqid=1, 256);
    // poll command complete
    sqHandle = nvme::getSQHandle(EC, funcHandle, sqid=1);
    ...
